01 — Problem

The Challenge

Analytical dashboards often require users to interpret large volumes of information quickly. The existing interface presented challenges around dense layouts, competing visual priorities, fragmented navigation, and dashboard scanning behaviour.

  • High information density
  • Unclear KPI prioritisation
  • Navigation overload
  • Weak visual hierarchy
  • Fragmented analytical workflows
02 — My Role

Scope of Work

I worked across interface redesign, workflow simplification, visual hierarchy refinement, and usability-driven dashboard improvements. The objective was to create an interface that supports analytical decision-making without overwhelming users.

03 — Goals

Design Goals

  • Reduce cognitive load
  • Improve dashboard scanability
  • Strengthen KPI visibility
  • Support analytical workflows
  • Create clearer navigation patterns
04 — Process

Design Decisions

Step 01 —

Information Architecture

Content grouping and dashboard organisation were refined to improve discoverability and reduce interface fragmentation.

Step 02 —

Visual Hierarchy

Priority metrics, analytical components, and navigation elements were redesigned using stronger hierarchy principles.

Step 03 —

Workflow Refinement

Layouts restructured to support faster interpretation and smoother movement between data views.

05 — Final Solution

Key Improvements

Hierarchy

Cleaner dashboard layout with improved KPI emphasis and reduced visual clutter.

Readability

Stronger analytics readability and clearer relationship between maps, charts, and metrics.

Navigation

More coherent navigation structure supporting smoother analytical workflows.

06 — Reflection

Takeaways

Designing analytical products requires balancing data density with usability. This redesign reinforced the importance of hierarchy, scanning behaviour, and workflow clarity within dashboard environments. It also highlighted how visual organisation can directly influence analytical comprehension and decision support.
